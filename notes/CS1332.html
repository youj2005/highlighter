CS 1332 - Data Structures and Algorithms

Big-O
  • Amortized Efficiency
    • Considers average Big-O instead of simply worst-case scenario
Array
  • Array - A contiguous sequenced collection of items (with indexes) of fixed length (capacity)
    • Stores primitives or objects
    • Statically allocated (fixed memory spots)
  • Declaration
    • obj[] arr = new obj[length];
    • obj[] arr = {item1, … };
  • Arrays can store references to objects
  • Accessing O(1)
    • Memory storage is contiguous (adjacent memory positions)
    • First index memory location is known
    • Data type is predetermined
    • new_address = start_address + i * data_size
  • Searching O(N)
    • Linear search for unsorted array
ArrayLists
  • ArrayList - A contiguous sequenced collection of items of variable length
    • Must be zero-aligned
      • First item must be in first memory location
  • Adding
    • At index O(N)
      • Must shift items afterwards backwards by one
    • At end O(1) (amortized)
      • Because no shifting
      • Non-amortized: O(N) because of resizing
  • Remove
    • At index O(N)
      • Must shift items afterwards forward by one
    • At end O(1)
      • No shifting
LinkedLists (LLs)
  • Linked List (LL) - A collection of nodes, each of which only connects to one other node (next), starting at the head
    • Doubly Linked Lists (DLL) - A LinkedList with pointer to the next and previous nodes
    • Singly Linked Lists (SLL) - A typical LinkedList with only the next pointer
    • Circularly Linked Lists (CLL) - A LinkedList where the head and the “tail” are the same
      • End-condition is now when current == head
    • Not necessarily contiguous
    • Check if head is null
    • Can contain tail pointer
  • Node - Stores data and a reference to the next node
  • Adding
    • At index O(N)
    • At front O(1)
      • Must reset head
    • At end O(N)
      • Must stop two nodes ahead
      • Unless if end pointer exists (O(1))
  • Removing
    • At index O(N)
    • At front O(1)
    • At end O(N)
      • Must check next next node
  • Edge Cases
    • Adding to back of empty SLL (create new node as header)
    • Removing back of SLL with size 1 (head = null)
  • Iterable/Iterator
    • Easily implements iterator by calling next for next()
Stacks
  • Stack - A data structure to remove the most recently added object (Last in First Out (LIFO))
    • Example behaviors
      • Undo/redo
      • Browser forward/backward
      • Recursive method (call stack)
  • Push - Adding to the top of the stack
  • Pop - Removing from the top of the stack
  • Peek - Looks at top object without removing
  • Implementations
    • Linked Stack - Singly LinkedList with the head being the most recent element
    • Array Stack - ArrayList starting at the end with most recent element as the last element
Queues
  • Queue - A data structure to remove the oldest object (First in First Out (FIFO))
    • Example Behaviors
      • Song queue
      • Waitlist
  • Enqueue - Adds to end of the queue
  • Dequeue - Removes the oldest in the queue (front)
  • Peek - Returns front of queue without removing
  • Implementation
    • Linked Queue - Singly LinkedList where head is the front of the queue
      • Enqueue → Add to tail
      • Dequeue → Remove head
    • Array Queue - Array with front and back indexes to keep track
      • Not ArrayList because both add and remove at beginning is O(N)
      • Front and back both start at start of array
        • Back wraps around to front when back reaches the end
      • Dequeue → increment first
      • Enqueue → increment last
  • Priority Queue - A data structure where the element with highest priority is removed
    • Data must be comparable
Deques
  • Deque - A double-ended queue with add/remove from front/back
  • Implementation
    • Array Deque
      • Same methods as Array Queue with addLast and removeFirst
      • Same wrap-around behavior
      • addFirst → decrement first
      • removeLast → decrement last
    • Linked Deque - Doubly LinkedList
      • Same methods as Linked Queue
      • addFirst → New head
      • removeLast → Doubly linked list so remove tail (O(N))
Trees
  • Tree - A linked hierarchical structure
    • Depth - Distance from the node to the root
    • Height - Distance from the node to the deepest sub-node
    • No cycles
  • Depth First Search (DFS) - Starts at root and traverses by going for deepest node
    • Preorder Traversal - Check current data first then recursively traverses through left child then right
      • Records top levels first
    • Postorder Traversal - Check current data after recursively traversing through left child then right
      • Records bottom levels first
    • Inorder Traversal - Check current data between recursively traversing through left child and right
      • Reads data in order
      • Not unique representation unlike pre and post
  • Breadth First Search (BFS) - Starts at root and traverses by searching across each depth
    • Levelorder Traversal - Add to a queue the root and then start looking at nodes from the front of the queue, enqueue their children, and then dequeue
  • Binary Tree - A tree where each node has only two children (left then right)
    • Balanced - If each kids heights differ by at most 1
    • Ability to recursively traverse by going to children
    • Binary Tree shapes
Binary Search Trees
  • Binary Search Tree - A tree where the left child is less than the node and the right is greater
  • Binary Search O(logn)
    • Implements a form of binary search but recursive
      • Depending on whether or not search number is larger/smaller than current node
      • Go to corresponding child
    • True/False
    • Degenerate trees have O(n)
  • Pointer Reinforcement - When recursively calling methods, set left/right to whatever the previous returns (always returns current node at end)
    • Add
      • Create new node if current is add
    • Remove
      • Use dummy node to store removed data, which at each step is the current data
      • Zero-Child Case
        • Return/set null
      • One-Child Case
        • Return/set left/right child
      • Two-Child Case
        • Change current node to have the data of successor/predecessor
          • Successor - Least number greater than current node (leftmost leaf in right child)
          • Predecessor - Largest number less than current node (rightmost leaf in left child0
          • Run helper method to get data of successor/predecessor
            • Helper is also pointer reinforced by returning the child of the succ/pred
          • Use dummy node to pass through helper and use the data from that (succ/pred) to change current data
Heap
  • Heap - A complete binary tree where each parent has a higher priority than its children
    • MaxHeap - A heap where parent > kids
    • MinHeap - A heap where parent < kids
  • ArrayHeap - Level order array, starting at index 1
    • For node at i
      • Left child: 2i
      • Right child: 2i+1
      • Parent: i/2
  • Heapify - Ensuring heap structure of complete heap represented by array by repeatedly swapping a node and its parent to maintain its relationship
  • Add O(logn)
    • Add data to end of array to ensure completeness
    • Up-Heap - Heapifying from bottom to top
  • Remove O(logn)
    • Remove data at beginning of array (root)
    • Copy last data to root
    • Down-Heap - Heapifying from top to bottom by finding highest priority child to swap with
  • Build Heap O(n)
    • Call down-heap on each sub-heap, starting at index size/2 (parent of rightmost leaf)
      • Climb upwards by one index and call down-heap on farther up sub-heaps containing other subheaps
HashMaps
  • Maps - Collections that are key-value paired, searchable, and unordered
    • Keys are unique
    • Put - Adds key and value to map
  • Hash-Map - A map where the key is hashed to represent its array index
    • Hashing - Converting object to integer where:
      • If two objects are equal, their hashcode is also equal
      • Not necessarily vice versa
      • Designed to avoid collisions
  • Collisions - When two values will be assigned the same key
  • Collision Avoidance
    • Resize
      • Re-assign key-value locations
    • Maximum Load Factor - Determines when you resize
    • Typical Max Load Factor = 3/5
  • Closed Addressing - Multiple values per key
    • External Chaining - Having a BST at each index
      • No duplicate keys
  • Open Addressing - One value per key
    • Soft Remove - When removing put in DEL marker
      • Do not include on resize
    • General Probing
      • When getting, probe to see when next occurrence is
        • Stop when hitting null/del
      • When putting, in priority order, put in desired spot, replace existing, in del marker, or null
    • Linear Probing - Upon collision look for empty slot by incrementing index
      • Primary Clustering/Turtling - Continuous blocks of data, which causes inefficiencies
      • Stop on reaching n-blocks
    • Quadratic Probing - Upon collision look for empty slot by incrementing by natural numbers squared
      • Secondary Clustering - Different keys collide in quadratic steps
      • To avoid infinite probing, resize upon probing n-times
SkipLists
  • SkipList - A sorted LinkedList where each node has a pointer that points to nodes ahead and to itself on another layer
    • Starts with -∞\infin∞﻿ and ends with ∞\infin∞﻿
  • Contains O(logn)
    • See if data is less than farthest node, if yes then jump down to look at closer nodes
    • If not, jump to that node and repeat
  • Add O(logn)
    • Find add spot through contains
    • Add to bottom and add levels depending on number of heads flipped
AVL
  • AVL - A binary tree that is forced to be balanced
    • Balance Factor - The height of left node - height of right node
      • Node is unbalanced if balance factor is not -1,0,1
  • Single (Left/Right) Rotation - Make one of the child nodes parents of unbalanced node
    • Have current node’s empty pointer adopt child of rotated child
    • Multiples rotations may be needed if left rotation and the swapped node is left-heavy and vice versa
      • Rotate subtree first
2-4 Trees
  • 2-4 Tree - A tree where a node can have 1-3 data points in sorted order
    • Number of children must be data + 1
    • Data must be sorted in ascending order counting the children
    • All leaves must have same depth
      • Guarantees O(logn) for all
  • Add
    • Always add to end of leaf
    • When overflowing, promote second/third data point to parent then split into two nodes
      • If parent overflows, also split parent
  • Remove
    • Case 1: Remove from leaf with multiple data
    • Case 2: Remove in middle of tree
      • Use predecessor/successor method (look rightmost/leftmost in node)
    • Case 3: Remove from leaf with one data
      • Pull down from parent and push one from neighboring leaf to parent
    • Case 4: Remove from leaf with one data (other leaves only have one data)
      • Fusion - Pull down from parent data and merge with neighbor
        • Fusion can propagate upwards
Sorting (Iterative)
  • Adaptive - Runs faster if data is partially sorted
  • Stable - Elements of equal value keep their order before/after
    • Can run multiple sorts
  • In-Place - Uses less than O(n) additional memory
Bubble Sort
  • Bubble Sort - Iterate through data and swap pairs if out of order and repeat till sorted
    • After first iteration, largest is always last, etc.
    • After each iteration, stop before completely sorted segment
    • Ω(n)\Omega(n)Ω(n)﻿, θ(n2)\theta(n^2)θ(n2)﻿, O(n2)\text{O}(n^2)O(n2)﻿
      • Worst case when reverse sorted
    • Adaptive, Stable, In-place
Insertion Sort
  • Insertion Sort - Iterate through data and place current data in previously sorted segment
    • Stability: insert duplicate afterwards
    • Ω(n)\Omega(n)Ω(n)﻿, θ(n2)\theta(n^2)θ(n2)﻿, O(n2)\text{O}(n^2)O(n2)﻿
      • Worst case when reverse sorted
    • Adaptive, Stable, In-place
Selection Sort
  • Selection Sort - Iterate through data and place current max at end of non-sorted segment
    • Ω(n)\Omega(n)Ω(n)﻿, θ(n2)\theta(n^2)θ(n2)﻿, O(n2)\text{O}(n^2)O(n2)﻿
      • Worst case when reverse sorted

CocktailShaker Sort

MergeSort

QuickSort
  • QuickSort - Given a pivot, place an element to the right/left of the pivot then recursively call on subarray

  • In-Place

Pattern Matching
Brute Force
  • Starting at start of both strings, increment through both to check if they have same character
    • If not, move pattern up one
  • O(mn)
Boyer-Moore
  • Last Occurrence Table
    • Create HashMap containing each letter as a key and the index of last occurrence as value
  • Search Algorithm
    • Align starts and start from back of pattern to check
    • 3 cases when conflict
      • Not in table
        • Move string all the way past current end index
      • Last occurrence of text character not encountered yet
        • Shift to match the index of the two characters
      • Last occurrence of text character encountered
        • Shift by one
    • When conflict, restart from back of pattern
  • O(mn),Ω(m+m/n)\text{O}(mn),\Omega(m + m/n)O(mn),Ω(m+m/n)﻿
KMP
  • Failure Table
    • Generate array of integers of length of pattern
    • Value at index is the length of substring that is a proper prefix as well as a proper suffix
  • Search Algorithm
    • Align starts and start from start of pattern to check
    • 2 cases when conflict
      • Start of pattern
        • Shift pattern forward one
      • Middle of pattern
        • Shift pattern to end of previous prefix defined in the failure table so that the start is aligned
    • When finding occurrence, jump over match
  • O(m+n),Ω(m+n)\text{O}(m + n), \Omega(m + n)O(m+n),Ω(m+n)﻿
