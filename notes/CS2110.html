    • Leftshift (<<) - Shifts everything left by _ digits

  • Masks - Constants you multiply a collection of bits by to do bitwise operations
    • CLEAR
      • Place 0s throughout and 1s where you want to keep the value
      • Use & operator
        • e.g. abcd & 0010 = 00c0
    • SET
      • Place 0s throughout and 1s where you want to set to 1
      • Use | operator
        • e.g. abcd | 0010 = ab1d
    • TOGGLE - Flips bit at certain index
      • Place 0s throughout and 1s where you want to toggle
      • Use ^ operator
        • e.g. abcd ^ 0010 = ab¬cd

    • First convert integer portion to binary
    • Convert float portion to binary by repeatedly multiplying by two
      • e.g. 0.75=0.5+0.25=2−1+2−20.75=0.5+0.25=2^{-1}+2^{-2}0.75=0.5+0.25=2−1+2−2﻿
        • 2∗0.75=1.52*0.75=1.52∗0.75=1.5﻿
        • Take the ones digit and that is the coefficient to the corresponding power of 2
    • Concatenate and remove leading one
  • Exponent

Transistors
  • Source - The “input” signal of a transistor, often power/ground
  • Drain - The “output” signal of a transistor, ground
  • NMOS connected to ground and PMOS connected to source
  • 3 different states
    • 0 volts - Grounded or designated so which represents the logical 0
    • 0-2.9 volts - Floating/High impedance state where it is neither logically 0 or 1
    • 2.9 volts - Designated as 2.9 volts which is a logical 1
  • NOT Gate
    • 0 → 1
    • 1 → 0
  • NOR Gate
    • 0 NOR 0 → 1
      • Anything else opens the p-types and closes at least one of the n-types
  • NAND Gate
    • 1 NAND 1 → 0
      • Anything else opens the n-types and closes at least one of the p-types
  • AND & OR Gates
    • Simply add a not gate to the NAND and NOR gates
  • Tri State Buffer - Allows data to only passed through if gate is true, floats if not

  • A AND B → AB
  • A OR B → A + B
  • NOT A → A‘
  • A XOR B

  • Decoder (n×2nn\times2^nn×2n﻿) - A switch that converts coded inputs into coded outputs with a one to one relation
  • Mutiplexor - A switch, using selecting bits, to select a single signal from an array of signals to an output
    • Basic multiplexor: 1 output, n control lines, 2n
      inputs

  • Subtractor

Sequential Logic
  • Sequential Logic - Stores elements so that output depends on input and state
  • R/S Latch
  • Gated D Latch
    • An R/S Latch that prevents invalid inputs (hence the notch which guarantees opposite inputs)
    • WE represents writability
  • Address Space - Number of addresses available
  • Addressability - Size of each memory location
  • Memory (address space x addressability) - Rows (address space) and columns (addressability) of Gated D Latches that stores bits
    • Decoders help decide which row we choose to write to
    • Multiplexers determine which row we take an output from by anding the row value with the output (if row = 0 then that is never getting outputted)
  • Clock
    • Alternates signal
  • Level-Triggered - A device that triggers depending on the state
    • e.g. Gated D Latch or R/S Latch
  • Edge-Triggered - A device that triggers due to the switching of a state
    • Leader-Follower/Flip Flop
      • Using the clock, allows data written to leader to be written to follower on edges
  • Register - A grid of leader/follower d-latches that stores concurrent operations
    • Stores data once every cycle
  • Finite State Machine - A machine that takes in current state and inputs to alter next state
  • Moore Machine - A machine that only depends on current outputs
  • Mealy Machine - A machine that depends on current outputs and inputs
LC-3
  • Von Neuman Model
LC3 Diagram
  • Bus (16)
    • Wire that delivers data to all comoponents
    • Only one instruction
  • Finite State Machine
    • Controls all the signals (everything not connected to a wire)
      • Combinational logic produces outputs (signals), next state (J), and COND & IRD, which control (J)
      • COND (3) - Depending on the value, sets the corresponding bit in J to 1
      • IR (1) - Passes through previous state regardles
  • General Purpose Register (GPR) (23×162^3\times 1623×16﻿)
    • Holds 8 addresses (R0-R7), each 16 bits
    • Destination Register (DR) - Specifies which register to save to
    • Source Register (SR) - Two output registers
      • SR1 and SR2 for operations with two numbers in ALU
  • Program Counter (PC) (16 bits)
    • Register that holds address of next instruction
    • Three Updates
      • Set to certain value by bus
      • PC ++
      • Add offset to PC value
  • Instruction Register (IR) (16 bits)
    • Holds value of the current instruction
  • Arithmetic Logic Unit (ALU)
    • Inputs: Data from register or immediate value from instruction
      • SR2MUX - Passes through immediate value or SR2
    • ALUK - Operation selector
      • 00: A + B
      • 01: A & B
      • 10: NOT A
      • 11: PASS A
  • Memory (216×162^{16}\times 16216×16﻿)
    • MEM.EN - controls access to memory
    • R.W - Controls read/write (0/1)
    • MDR - Takes in data to write and outputs data stored at memory location
    • MAR - Gives memory location to read/write
  • Condition Code Register
    • Stores whether or not the last evaluation was negative, neither, or positive
Macrostates
  • Operations go through all three macrostates to properly execute
  • Fetch (3 clock cycles)
    • 1st cycle: Takes incremented PC and puts it in MAR.
    • 2nd cycle: Put data into MDR
    • 3rd cycle: Take data from MDR (through bus) and store in IR
  • Decode (1 clock cycle)
    • Takes in instruction from IR and uses opcode to determine operation
  • Execute (variable clock cycle)
    • Executes instruction through signals
Opcodes
  • The four most significant bits in an instruction that indicates what operation to execute
  • Arithmetic (AND & ADD)
    • 5th bit determines SR2MUX
    • Register - Pulls from register
    • Immediate - Pulls from output
  • Load (memory→register)
    • PC Relative (LD) - mem[PC* + PCOffset9]
      • PC* is iterated PC
      • Only +- 256 from PC
    • Indirect (LDI) - mem[mem[PC* + PCOffset9]]
    • Base + Offset (LDR) - mem[BaseReg + offset6]
      • Anywhere
  • Store (register→memory)
    • Same as LD except opposite direction
  • Control
    • Branch (BR) - Branches to a different memory location for PC depending on NZP value
      • If one of the values are both true, then execute
      • PC* + PCoffset9 → PC
    • Jump - Always jumps to different memory location for PC

I/O
  • Synchronous - Data transferred at fixed rate (read/write every x cycles)
  • Asynchronous - Data transferred on signal
  • Polling - Processor repeatedly checks if data is ready to transfer
  • Interrupt - Device interrupts processor to transfer
  • Input
    • KBDR (Keyboard Data Register) - Last eight bits [7:0] of register that contains ASCII character
      • 0xFE02 in memory
    • KBSR (Keyboard Status Register) - Bit 15 [15] of register that sets when new data can be read (0 → false, 1 → true)
      • 0xFE00 in memory
    • Assembly code
      • Keep loading KBSR and when it becomes negative (15 bit turns 1), load KBDR
  • Output
    • DDR (Display Data Register) - Last eight bits [7:0] of register that contains ASCII character
      • 0xFE06 in memory
    • DSR (Display Status Register) - Bit 15 [15] of register that sets when new data can be displayed (0 → false, 1 → true)
      • 0xFE04 in memory
    • Assembly code
      • Keep loading DSR and when it becomes negative (15 bit turns 1), write into DDR
PSR
  • PSR (Processor Status Register) - Stores information about privilege, priority, and current condition codes
    • Privilege [15]
      • Supervisor Mode (0) - Mode that operating system executes in (access to everything)
      • User Mode (1) - Mode that user operates in (cannot access I/O space or system space)
    • Priority [10:8]
      • PL0-PL7 determines order of execution
        • Higher number is higher priority

Caller & Callee
  • Caller
    • Caller Buildup
      • Push arguments into stack in reverse order (use R6)
      • Jump to subroutine
    • Caller Teardown
      • Load return value and pop (from R6)
      • Pop arguments (from R6)
  • Callee
    • Callee Buildup
      • Reserve space for return value, and store R7 and R5
      • Set R5 to R6 incremented
      • Reserve space for local variables and store old registers (from R6)
    • Callee Teardown
      • Store return value (from R5)
      • Restore register values and pop local variables (from R6)
      • Pop and restore R5 and R7 (from R6)

Dynamic Memory
Heap Functions
  • Dynamic Memory - Memory allocated during runtime to adjust for needs
  • malloc(size)- Allocates block of memory on heap of size and returns a void pointer
    • e.g. int* pointer = (int*) malloc(5*sizeof(int));
    • returns NULL if heap has no more space
  • free(pointer) - Frees up memory at pointer (including everything within size)
  • realloc(pointer, size) - Changes the size of the allocated memory at pointer
    • returns void pointer
    • Keeps data the same
    • If size is larger, everything past previous size is indeterminate
  • calloc(num, size)- Allocates (num*size) memory and initializes each value to 0
    • returns void pointer
Usage
  • When referencing pointers in functions, due to teardowns in the stack, data may be lost
    • Allocate space on heap and return a pointer to the heap to preserve data
